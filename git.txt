* Git *

=> Theory : 
  
  -> Git is a distributed version control system.
  -> It' s system that records changes to our files over time.
  -> We can recall any specific version of those files at any time.
  -> many people can easily collaborate on single project.


=> Setup and Commands:

1) Download git and install it in your system
2) git --version
3) set git name and email using

    git config --global user.name 'name'
    git config --global user.email 'email'

    OR if(locally set karvu hoy to)

    git config user.name 'name'
    git config user.email 'email'

    ->To check user name and email: 

        git user.name
        git user.email

4) initialize git in folder.....that create the repostory in our folder(.git)
     git  init

      -> When we fire this command it will also create a new defaultly active master branch

5) to check modified files..
     git status (modified files staging area ma ny hoy to red btavse... hse to green)

6) to add files in stagging area...
     git add 'file name'    
     git add . (bdhi j files ne staging area ma eksathe put karva)    

7) to remove files from stagging area...
     git rm --cached 'file name'  (single file ne remove karva)

     OR

     git reset HEAD -- . (bdhi files ne remove karva)

8) to commit our work (basically is a save point of our work)
     git commit  -m "description is work"

     OR 

     git commit -a -m 'message'

     OR 

     git commit --amend  (agal ni commit ma changes save karva hoy to)

9) to show commit history
     git log
     git log --oneline (to show commit history in shorthand way)

10) undoing things
  
  ->There are three methods to Undoing things: 

    i)Chekout commit:

      ->git log --oneline

        ->This command is used to just see the perticular commited code using their commit id.

      ->git checkout 'id' 

        ->The changes made with this command are read only

    ii)Revert commit:  

      ->git revert 'id' 

        ->This command is used to revert at any commiteed point.
        ->Now if we check our commit's using git log --online, we see that we not acctually deleted that perticullar commit but we just make a new commit whitout that perticular code which we reverted.

    iii)Reset commit:

      ->git reset 'id'

        ->after running this command if we check our commit we see that all the above commit of that particular id was deleted
        ->But the changes in our code are still same. If we want also delete those code changes with that commit than run

          git reset 'id' --hard

  
11) braches
       git branch 'name' (create new branch)
       git branch -a (to show all branch)
       git checkout 'branch name' (to change branch)
       git brach -D 'branch name' (to delete branch...jyare branch master ma merge n kreli hoy tyare)
       git brach -d 'branch name' (to delete branch...jyare branch master ma merge kreli hoy tyare)
       git checkout -b 'name' (shorthand way to create and switch the branch)

=> branch merge karva pela j branch ma merge karvu hoy ema switch thvu
      git merge 'name' (to merge branch)

=> Renaming Git Branch
  
      git checkout <old_name>
      git branch -m <new_name>
      git push origin -u <new_name>
      git push origin --delete <old_name>

12) Discard all local changes, but save them for possible re-use later: 
       git stash

13) Discarding local changes (permanently) to a file: 
      git checkout -- <file>

14) Discard all local changes to all files permanently:  
      git reset --hard

15) For fetch all newly created branches
       git fetch --all
