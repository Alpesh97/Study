*redux and contex diff*

	-> normally banne application no data store karva mate use thay.
	-> for an example apdi site ma 4 component che 
		header -> sidebar -> main content -> footer
		hve koi user or koi bija component ma main content no data ni jarur pdi to jo redux no use karyo to e su karse hierarchy follow karse... directly main content no data ny aape....jyare contex hook api no use kri ne aapde main content no data or property directly use kri sakiye.

		OR

	-> 2 or vdhare component vachche communicate karva mate use thay em pn kai sakiye.

	ex.- koi ecommerce ni site ma main-content na component mathi koi vastu cart ma add kri to have ene related change cart na component ma pn karva pde to e bdhu karva mate redux and contex no use thay


	=>contex:

		->React hooks provide a concept call contex. 
		means koi hierarchy che ex. A->B->C->D to hve aapde A parent ni koi property ne khali D child ma j use karvi hoy to eva case ma contex hook no use thay.
		->React contex API allows us to easily access data at different level of the component tree. without passing prop to every level.

		->there are three steps to create a contex:

			1) createContext() : create a contex
			2) provider : bija use kri sake etle ene provide karvu
			3) consumer : provider ma data provide thto hoy tene consume(leva) mate

	
	=>redux:

		->A->B->C->D to hve aapde A parent ni koi property ne khali D child ma j use karvi hoy to redux su karse k pela e B ma jse pasi C ma pasi D ma use kri sakisu.

=>How to styles an elements in react app :
	
	->Apde koi pn normal web app ma style kriye e rite react ma pn koi .css file create krine style lakhi sakiye ane e style file ne jya use krvi che tya just import karvi

		ex.-

			style.css

					backgroud-color: red,
					padding: 10px, OR
					padding: 10,
				}

			app.js

				import './style.css';

				import React from 'react';

				export default function App() {

				    return (
				      <div className="root">
				        
				      </div>
				    );

				}	

	->But in almost all react app when we use material ui we will be using the *jss syntax* to define css.

		->Official documentation:

			https://cssinjs.org/from-sass-to-cssinjs

		->basically it allow us to define these css rule with the help of javascript.
		->to use this we have to install some of the plugins for that but when we create our react app some packages are default installed in our node_module file and you can check it with starting name *jss-plugin-*.
		->these packages allow us to work with jss syntax in our react app.

		->Let's see how to write jss syntax in our component file:

			ex. - 

				const useStyles{
				  root:{
				    backgroudColor: 'red',
					padding: '10px',
				  }
				}	

				-> this is how we define css with jss syntax:
				-> now we have to convert this to it's original syntax. To do this convertion there two methods in material ui

				1) makeStyles :

					import { makeStyles } from "@material-ui/core";

					const useStyles = makeStyles({

					  root:{
					    backgroudColor: 'red',
						padding: '10px',
					  }

					});


					export default function App() {

						const classes = useStyles();

					    return (
					      <div className={classes.root}>
					        
					      </div>
					    );
					}

					->here *useStyle* is a constant object....aapde koi pn name rakhi sakiye

				2) withStyles :

					import { withStyles } from "@material-ui/core";

					const useStyles = {
					  root:{
					    backgroudColor: 'red',
						padding: '10px',
					  }
					}


					const App = (props) => {

						const {classes} = props;

					    return (
					      <div className={classes.root}>
					        
					      </div>
					    );
					}

					export default withStyles(useStyle)(App);


					->here *useStyle* is a constant object....aapde koi pn name rakhi sakiye

=>Component customization :
	
	->We can customization component using two way in material ui:	

		1)className attribute :

			->It is same as *How to styles an elements in react app*


		2)classes attribute :

			->Material ui na koi component no to aapde easily class api sakiye but...e component jyare html ma render thse tyare jo eni andar bija extra div or koi bija tag auto generate thta hoy to tene class aapva mate aa method no use thay che

			ex. - 

				import { makeStyles } from "@material-ui/core";
				import AccessAlarmIcon from '@material-ui/icons/AccessAlarm';

				const useStyles = makeStyles({

				  btnWapper:{
				    backgroundColor:'red'
				  },
				  btnInner:{
				    backgroundColor:'green'
				  },

				});


				export default function App() {

					const classes = useStyles();

				    return (

				      <IconButton classes={{root:classes.btnWapper,label:classes.btnInner}}>
                		<AccessAlarmIcon />
                	  </IconButton>

				    );
				}	

				where, root = j tag ni pasal -root lagtu hse ene lagse
					   label = j tag ni pasal -label lagtu hse ene lagse

				o/p: https://prnt.sc/1098087

				*note : 

					->aa method thi khali class define karya hoy ane css n lakhi hoy to class pn html ma show ny kre.
