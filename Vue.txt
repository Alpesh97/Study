3 types of mobile app:-

	=>Web Apps
		->Built using Html/CSS/JavaScrip
		->Run inside a browser
		->Built like a regular website
		->Designed to look good in mobile devices
		->Used in all platform like android,ios,blackberry.

		Disadvantages:
			->Need a browser to run
			->Slower than native apps
			->Can not submitted to app store or play store

	=>Native Apps
		->Most Common type of mobile app
		->Built for specific platform that's why it is very fast
		->Built using native programming languages ex.-
			Android app coded in Java it uses the SDK for that platform
			iOS app codded in Swift or sometime objective-c.
		->Interact with device utilities(any feature of the phone like:carema location,storage,etc.)

		Disadvantages:
			->Single application (means Java ma bnaveli androin app android device ma j work karse)

	=>Hybrid Apps
		->Combination of Web and Native apps
		->Used HTML/CSS/JavaScript. So it's easy to build
		->Single app for all platforms
		->No browser needed
		->Interact with device utilities(any feature of the phone like:carema location,storage,etc.)

		Disadvantages:
			->Slower than Native apps (because native app single platforms mate j bne che)


MVC framework:-
	->Is an architectural pattern that separates the application layer into Model,View and Controller
	means organized way to write a code
	->In this we divide whole code into three part:

		Modal: In this we perform *Database* oparation.

		View: In this we create *Design* (HTML/CSS)

		Controller: In this we write our *Logic*(c= a + b) and it Provide Link between Modal and View.


What is DOM:-
	->When a web page is loaded, the browser creates a *DOM* of the page and In that elements are organized as a tree structure.
	->the DOM is just another way to represent your HTML document

	where D = Document (means aapdu aakhu html page),
		  O = Object (body,head,main,etc. aa bdha tag),
		  M = Modal ()

What is virtual DOM:-
	->A virtual DOM only look at the differences between the previous and current HTML and changes the par that required to be updated.(meand HTML ma koi changes karsu to dom aakhu nvu ny bane..just j changes karya hse tya j update karse)
	->It updates faster.
	->Canâ€™t directly update HTML
	->Updates the JSX if element updates
	->DOM manipulation is very easy
	->No memory wastage

What is real DOM:-
	->This will update the entire tree structure of HTML tags until it reaches the data to be updated(means koi ek jagyaye change karyo hse to aakhu dom tree pelethi banse)
	->It updates slow
	->Can directly update HTML
	->Creates a new DOM if element updates
	->DOM manipulation is very expensive
	->Too much of memory wastage
